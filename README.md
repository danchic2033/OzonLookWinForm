# Оконное приложение - магазин одежды (WinForms)

## Обзор системы

Данная система представляет собой прототип приложения для онлайн-покупки одежды с возможностью примерки по фото. В будущем планируется внедрение модели по примерке одежды на фото пользователя, что делает проект актуальным для массового потребителя, желающего дистанционно подбирать образы и просматривать историю покупок и избранное.

## Используемые технологии

- **C# WinForms** — для разработки пользовательского интерфейса.
- **Guna.UI2.WinForms** — сторонняя библиотека для стилизации элементов интерфейса (кнопки, панели, изображения).
- **Entity Framework Core** — для работы с базой данных (поддержка CRUD операций, хранение информации о товарах).
- **MySQL** — в качестве базы данных, подключенной через EF Core.
- **JSON-файл (`appsettings.json`)** — для хранения строки подключения к базе данных.
- **Объектно-ориентированное программирование** — для моделирования данных и логики (классы `Product`, `Photo`, `FavoriteProducts`, `HistoryProducts` и др.).

## Основные функции

- Просмотр каталога товаров с автоматическим формированием отображения на основе данных, предварительно спарсенных из онлайн-магазина.
- Возможность выбрать понравившийся товар и сразу примерить его (функция-заготовка).
- Добавление товаров в избранное.
- Просмотр истории просмотров и примерок.
- Управление приложением осуществляется из главной формы (`MainForm`) с помощью кнопок и вкладок, реализованных через `UserControl`.
- Каталог поддерживает прокрутку и динамическое отображение элементов.

## Логика приложения и принципы алгоритмов

- **Модель данных** реализована через классы `Product`, `Photo`, `FavoriteProducts`, `HistoryProducts`, которые хранят информацию о товарах, фотографиях и связанной с ними истории и избранным.
- **Каталог** формируется динамически при запуске, с помощью метода `InitCatalogMap`, который создает кнопки и изображения для каждого товара, размещая их в сетке.
- **Фильтрация** товаров осуществляется по выбранным категориям (мужская/женская одежда, тип одежды) через LINQ-запросы.
- **Обработка событий** осуществляется через назначение обработчиков на кнопки, что позволяет отображать подробную информацию о товаре, добавлять его в избранное или историю просмотров.
- **Автоматизированное отображение каталога** — реализовано динамическим созданием элементов (`CatalogItem`) и прокруткой с помощью компонента `Guna.UI2.WinForms`.
- **Данные для каталога** — спарсены заранее из онлайн-магазина и загружены из папок с изображениями, что обеспечивает быстродействие и удобство тестирования.

## Особенности и преимущества системы

- **Автоматизированное отображение каталога** — реализовано с помощью динамического создания элементов управления (`CatalogItem`) и прокрутки.
- **Динамическое управление интерфейсом** — переключение вкладок осуществляется через метод `ShowControl`, который показывает нужный `UserControl`.
- **Модульность и расширяемость** — реализована за счет отдельной логики для работы с избранным, историей и каталогом.
- **Оптимизированное использование данных** — все изображения загружены из папок, что позволяет быстро отображать контент без задержек.

## Планы по развитию

- Внедрение нейронной модели для примерки одежды — создание и интеграция нейронной сети, которая сможет автоматически примерять одежду на фото пользователя, делая процесс максимально реалистичным и автоматизированным.
